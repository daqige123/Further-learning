# localStorage 与 indexDB 的区别

localStorage 是浏览器提供的  **同步 key-value 存储** ，只能存字符串，容量大约 5MB，适合存储简单配置或 token。

indexDB 是浏览器内置的 nosql，支持复杂对象储存，索引查询，事务操作，并且api 是异步的，所以不会阻塞主线程，适合储存大量结构化数据或者缓存。

# 其他存储方式

| 特性           | cookie         | localStorage | sessionStorage | IndexedDB    |
| -------------- | -------------- | ------------ | -------------- | ------------ |
| 存储位置       | 浏览器         | 浏览器       | 浏览器         | 浏览器数据库 |
| 生命周期       | 可设置过期时间 | 永久         | 页面会话结束   | 永久         |
| 容量           | 4KB 左右       | 5MB 左右     | 5MB 左右       | 几十 MB 以上 |
| 是否随请求发送 | ✅ 会          | ❌ 不会      | ❌ 不会        | ❌ 不会      |
| API 类型       | 同步           | 同步         | 同步           | 异步         |
| 数据类型       | 字符串         | 字符串       | 字符串         | 任意对象     |
| 典型场景       | 登录态         | 本地缓存     | 临时状态       | 大数据存储   |

## cookie：

主要是用于于服务端通信, 常用于，登录状态、session id用户身份认证。特点是：

1. 容量小，
2. 每次http请求，都会携带上,
3. 可以设置过期时间。

## sessionStorage：

和 localStorage 很像，但生命周期不同：页面一关闭，数据就清空， 不同 tab 不共享。
